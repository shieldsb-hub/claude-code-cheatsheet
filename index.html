Claude Code & GitHub

The simplest possible guide to talking with Claude in your terminal — and using it to commit and push code to GitHub.

1 What is Claude Code?

Claude Code is an AI coding assistant that lives in your terminal. You chat with it like a teammate, and it reads your files, runs commands, edits code, and uses git for you.

Install it once

Open Terminal and paste this:

$ npm install -g @anthropic-ai/claude-code

Then start it from any folder:

$ cd ~/Desktop/my-project
$ claude
Tip: The first time you run it, it will ask you to sign in. Just follow the link.

2 How to talk to Claude

Just type plain English. Be specific about what you want.

✗ Vague

> fix the bug

✓ Specific

> the login button on index.html
   doesn't redirect after click —
   please find and fix it

Useful slash commands

/helpShow all commands
/clearStart a fresh conversation
/initMake Claude read your project and create a CLAUDE.md memory file
/reviewReview your current changes
EscStop Claude mid-response

3 GitHub setup (one time)

Before pushing code, make sure git and GitHub know who you are.

Install the GitHub CLI

This is the easiest way to log in. Claude Code uses it under the hood.

$ brew install gh           # mac
$ gh auth login             # follow the prompts

Tell git who you are

$ git config --global user.name "Your Name"
$ git config --global user.email "you@email.com"

4 The everyday GitHub flow

This is the loop you'll repeat all day. Claude does almost all of it for you.

  1. Make changes — ask Claude to write or edit code. 
    > add a dark mode toggle to the header
  2. Check what changed 
    > show me what's changed

    Claude runs git status and git diff and explains it in plain English.

  3. Commit — just ask. 
    > commit these changes

    Claude reads the diff, writes a clean commit message, and runs git commit for you. It will show you the message first and ask before committing.

  4. Push to GitHub 
    > push to github

    Done. Your code is on GitHub. If the repo doesn't exist on GitHub yet, ask Claude to "create a github repo and push it" — it will use gh repo create.

  5. Open a Pull Request (when working on a branch) 
    > open a PR to main with a good
   title and description
Remember: Claude always asks before doing anything destructive (force push, deleting branches, etc). When in doubt, just answer "no" and rephrase your request.

5 Start a brand new project

From empty folder to "live on GitHub" in one conversation.

$ mkdir my-cool-app && cd my-cool-app
$ claude

> initialize a git repo, build me a simple
   landing page with HTML and CSS, then create
   a github repo called "my-cool-app" and push it

Claude will:

  • Run git init
  • Write the files
  • Make the first commit
  • Run gh repo create and push

7 A few safety rules

  • Read before you press Y. Claude shows you commands before running them — actually look.
  • Commit often. Small commits are easier to undo than big ones.
  • Never paste secrets. Don't put API keys or passwords into the chat.
  • Use branches for anything bigger than a tiny fix. git checkout -b feature/thing
